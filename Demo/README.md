# AirStrings React Native Demo

A single-screen app that consumes `@airstrings/react-native` (installed from the packed tarball) against a fully local stack: Postgres + MinIO + the Go backend. It renders live signed strings, ICU plurals, locale switching, and demonstrates that a tampered bundle is rejected.

## One-time setup

```bash
make setup        # stack (docker + backend) -> seed (strings + bundles) -> config (keys + seed fixtures)
```

`make setup` runs three steps you can also run individually:

- `make stack` — start Postgres + MinIO, apply migrations, provision the bucket, run the backend on `:8080`.
- `make seed` — sign up / log in, create the demo project + 7 strings, publish `en`/`fr`/`es` bundles, write `.demo-env`.
- `make config` — write `demo.config.generated.ts` and download the published bundles into `airstrings/bundles/` as offline seed fixtures.

## Run

```bash
make ios          # iPhone 17 Pro simulator
make android      # boots the emulator if needed, wires adb reverse for :8080 and :9000
```

In the app: switch locales, press **Refresh**, and watch the status line (`locale · rev · ready`) and the log pane.

## Tamper check

```bash
make tamper       # mutates the published en bundle in MinIO
```

Then press **Refresh** in the app: the SDK rejects the bundle with `SIGNATURE_VERIFICATION_FAILED` (shown in the log pane) and keeps serving the last good strings. Run `make seed` to restore a clean, freshly signed bundle.

## Teardown

```bash
make stack-down   # stop the backend and the docker services
```

## Notes

- `demo.config.generated.ts`, `.demo-env`, and `airstrings/bundles/*.json` are generated by the scripts and git-ignored — run `make setup` after a fresh clone.
- After rebuilding the SDK, re-pack it (`npm run build && npm pack` in the SDK root) and reinstall the tarball here.
- Plural rows render as the raw ICU pattern on the default Hermes engine (no `Intl.PluralRules`) — see "ICU formatting on Hermes" in the SDK README. This demo ships no polyfills so the raw fallback is visible.
